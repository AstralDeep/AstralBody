# Quickstart: Apple Clients Production Release (053)

A per-user-story **validation runbook** — how to prove each slice works end-to-end and
maps to its success criteria. Mirrors [051/quickstart.md](../051-apple-native-clients/quickstart.md),
extended for the production (signed / submitted) posture.

All paths are repo-relative to the AstralBody root. Client work runs on a Mac with
**Xcode 16+** (project `objectVersion 77`). Steps that consume **operator-supplied signing
material, store metadata (listing copy + screenshots), or realm changes** are marked
**[OPERATOR-GATED]** — the surrounding work proceeds in parallel and the live
upload/submit is blocked until those are present (see spec Assumptions, research D1/D19).
Icon artwork is **no longer a gate**: the operator supplied the master and every Apple icon is
already generated by a committed, dependency-free script (research D15; verified in US8).

## Shared setup

```bash
# Project + schemes (research D9: AstralWatch shared scheme must exist)
PROJ=apple-clients/AstralApp/AstralApp.xcodeproj
xcodebuild -list -project "$PROJ"           # expect schemes: AstralApp, AstralWatch

# Core logic + the server→client drift guard (any Mac, no signing needed)
swift test --package-path apple-clients/AstralCore     # expect all green, incl. ManifestDriftTests

# Backend the clients target (dev posture for local checks)
docker compose up -d                                   # postgres + astralbody
```

Identities in play everywhere (research D8): bundle family `com.personalailabs.astraldeep`
(+`.watch`), URL scheme `com.personalailabs.astraldeep`, OAuth clients `astral-mobile`
(iOS) / `astral-desktop` (macOS) / `astral-watch`, realm `https://iam.ai.uky.edu/realms/Astral`,
backend `https://sandbox.ai.uky.edu`.

## US1 — Signed, App-Store-validatable archives (iOS+embedded-watch, macOS) (P1 → SC-001/SC-001a/SC-005a)

**Proves**: the **two** platform builds — iOS (with the watch app embedded) and macOS —
archive, export app-store, and pass App Store Connect validation with zero signing/compliance
errors (FR-001..FR-008). There is no third archive: the watch rides inside the iOS build (D12/D19).

**Prerequisites**

- **[OPERATOR-GATED]** Apple Team ID; Apple Distribution certificate + App Store provisioning
  **three** App Store profiles — iOS + macOS (`com.personalailabs.astraldeep`) AND watchOS (
  `com.personalailabs.astraldeep.watch`) imported into the login keychain (or CI temp keychain,
  D1); the **single** Universal Purchase App Store Connect record (iOS + macOS platform versions,
  D19); an App Store Connect API key (`.p8` + key id + issuer id).
- Icon assets are **already generated** by `apple-clients/Scripts/generate_app_icons.py` from the
  supplied master — no operator icon step remains (see US8; D15).
- `apple-clients/ExportOptions-ios.plist` and `apple-clients/ExportOptions-macos.plist` present, each with `method = app-store`.

**Actions**

```bash
PROJ=apple-clients/AstralApp/AstralApp.xcodeproj
OUT=build/053-archives
ICONSET=apple-clients/AstralApp/AstralApp/Assets.xcassets/AppIcon.appiconset

# 1) Icon set is present and correctly formed BEFORE archiving (SC-005a; see US8 for generation)
python3 apple-clients/Scripts/generate_app_icons.py --check    # asserts slot sizes, iOS/watch opacity, macOS gutter
sips -g hasAlpha "$ICONSET/AppIcon-1024.png" \
     apple-clients/AstralWatch/Assets.xcassets/AppIcon.appiconset/AppIcon-1024.png   # expect hasAlpha: no (alpha => ITMS-90717)
sips -g hasAlpha "$ICONSET/mac-512x512@2x.png"                 # expect hasAlpha: yes (rounded-rect gutter is intentional on macOS)

# 2) Archive the TWO platform builds — iOS carries the embedded watch app; macOS is separate (D19)
xcodebuild archive -project "$PROJ" -scheme AstralApp \
  -destination 'generic/platform=iOS' \
  -archivePath "$OUT/AstralApp-iOS.xcarchive" \
  CURRENT_PROJECT_VERSION="$(date +%s)"        # build number injected as in the pipeline, D5

xcodebuild archive -project "$PROJ" -scheme AstralApp \
  -destination 'generic/platform=macOS' \
  -archivePath "$OUT/AstralApp-macOS.xcarchive"

# 2a) Confirm the watch app is EMBEDDED inside the iOS archive — no separate watch archive/record (D12/D19)
ls "$OUT/AstralApp-iOS.xcarchive/Products/Applications/AstralApp.app/Watch/"   # expect the embedded AstralWatch.app
#    (the watch also compiles standalone by scheme name in US3's clean-clone matrix)

# 3) Export app-store artifacts (two, not three)
xcodebuild -exportArchive -archivePath "$OUT/AstralApp-iOS.xcarchive" \
  -exportOptionsPlist apple-clients/ExportOptions-ios.plist \
  -exportPath "$OUT/export-iOS"
xcodebuild -exportArchive -archivePath "$OUT/AstralApp-macOS.xcarchive" \
  -exportOptionsPlist apple-clients/ExportOptions-macos.plist \
  -exportPath "$OUT/export-macOS"

# 4) Validate against App Store Connect  [OPERATOR-GATED on API key]
xcrun altool --validate-app -f "$OUT/export-iOS/AstralApp.ipa" --type ios \
  --apiKey "$ASC_KEY_ID" --apiIssuer "$ASC_ISSUER_ID"
xcrun altool --validate-app -f "$OUT/export-macOS/AstralApp.pkg" --type macos \
  --apiKey "$ASC_KEY_ID" --apiIssuer "$ASC_ISSUER_ID"

# 5) Confirm compliance settings baked into the signed binaries
codesign -dvvv "$OUT/export-macOS/AstralApp.app" 2>&1 | grep -i 'flags.*runtime'   # Hardened Runtime
codesign -d --entitlements - "$OUT/export-macOS/AstralApp.app" \
  | grep -i 'app-sandbox\|network.client\|keychain'                                 # App Sandbox + minimal client entitlements (D7)
plutil -p "$OUT/AstralApp-iOS.xcarchive/Products/Applications/AstralApp.app/PrivacyInfo.xcprivacy"   # privacy manifest present (D4)
```

**Expected outcome**

- Both `--validate-app` runs report **no** missing-icon, missing-privacy-manifest, ATS,
  entitlement, or signing errors (SC-001).
- The iOS archive **contains the embedded watch app** (`…​/AstralApp.app/Watch/AstralWatch.app`),
  which carries its own icon, export-compliance declaration, and provisioning profile — the
  combined archive validates as one (US1 acceptance 3; D12/D19).
- macOS binary shows **Hardened Runtime** flag and an **App Sandbox** entitlement with only
  `network.client` + keychain access — no temporary-exception entitlements (FR-003, D7).
- The App Store icon slots pass mechanically: the iOS/watch 1024 icons report `hasAlpha: no` and
  the macOS slots report `hasAlpha: yes` (SC-005a, generated + checked in US8); `PrivacyInfo.xcprivacy`
  parses and `ITSAppUsesNonExemptEncryption=false` is set for both app and watch (FR-005/FR-007).
- Two back-to-back archives of the same target carry **distinct, increasing** build numbers
  with no manual edit (FR-008; re-run step 2 and compare `CFBundleVersion`).
- **[OPERATOR-GATED]** live submit-for-review is exercised by US4's pipeline; the manual
  action equivalent is `xcrun altool --upload-app …` then the App Store Connect
  submit action (SC-001a).

## US2 — Production endpoint via build-time config, no hardcoded dev host (P2 → SC-002/SC-003)

**Proves**: a Release build resolves its backend + realm from xcconfig/Info.plist (default
sandbox), repoints with no code edit, the watch (now an embedded companion) overrides at runtime
*and falls back when no companion is installed*, and no dev endpoint is reachable in Release
(FR-009..FR-011, FR-011a, D2/D3/D12).

**Prerequisites**: none beyond Shared setup. No operator gating.

**Actions**

```bash
# 1) Build a Release iOS app and read its RESOLVED endpoint from the built Info.plist
xcodebuild build -project "$PROJ" -scheme AstralApp -configuration Release \
  -destination 'generic/platform=iOS' -derivedDataPath build/rel-ios
APP=build/rel-ios/Build/Products/Release-iphoneos/AstralApp.app
plutil -extract ASTRALServerBaseURL raw "$APP/Info.plist"     # expect https://sandbox.ai.uky.edu
plutil -extract ASTRALKeycloakAuthority raw "$APP/Info.plist" # expect https://iam.ai.uky.edu/realms/Astral

# 2) Repoint with NO source change — edit only the xcconfig value, rebuild
#    (edit apple-clients/Config/Release.xcconfig ASTRAL_SERVER_BASE_URL, then re-run step 1)

# 3) ATS: Release must NOT carry a blanket arbitrary-loads exception (D3/FR-006)
plutil -p "$APP/Info.plist" | grep -i 'NSAllowsArbitraryLoads' && echo "FAIL: ATS exception in Release" || echo "OK: ATS clean"

# 4) Grep the Release-reachable source/config for any hardcoded dev endpoint (FR-010/SC-003)
grep -rniE 'localhost|127\.0\.0\.1|:8001|#if DEBUG.*serverBase' \
  apple-clients/AstralCore/Sources apple-clients/AstralApp apple-clients/AstralWatch \
  --include='*.swift' | grep -viE '(Debug\.xcconfig|#if DEBUG)'   # expect NO Release-reachable hits

# 5) Watch is an EMBEDDED companion, not watch-only (FR-011a): confirm the converted Info.plist
plutil -p apple-clients/AstralApp/WatchInfo.plist \
  | grep -iE 'WKWatchOnly|WKCompanionAppBundleIdentifier|WKRunsIndependentlyOfCompanionApp'
#    expect: NO WKWatchOnly; WKCompanionAppBundleIdentifier = com.personalailabs.astraldeep;
#            WKRunsIndependentlyOfCompanionApp = YES

# 6) Override is OPPORTUNISTIC over WatchConnectivity, never a dependency (D12/FR-011)
grep -rn 'serverBaseURL\|serverOverride\|WCSession\|isCompanionAppInstalled' \
  apple-clients/AstralWatch apple-clients/AstralCore
#    expect: the override reads the companion-pushed value ONLY when isCompanionAppInstalled == true,
#            and falls back to the xcconfig default otherwise (no App Group entitlement, D7)
```

**Manual (on device, cross-ref US6) — BOTH override paths**: with the iPhone companion installed
and a server override set, confirm the watch targets the chosen endpoint; then with the companion
**absent** (`isCompanionAppInstalled == false`), confirm the watch falls back to the build-time
default and still signs in via QR device-login — the override never blocks the watch.

**Expected outcome**

- Default Release build targets `sandbox.ai.uky.edu` + realm `iam.ai.uky.edu/realms/Astral`
  (FR-009, SC-002 first-attempt sign-in verified in US6).
- Changing only the xcconfig value repoints the app; **no `.swift` edit** (FR-009 acceptance 2).
- No `NSAllowsArbitraryLoads` in the Release Info.plist; the localhost exception is
  Debug-scoped only (`Config/Debug.xcconfig` / `NSAllowsLocalNetworking`, FR-006).
- Grep returns **zero** Release-reachable hardcoded dev endpoints (SC-003); any `localhost`
  hit is inside a `#if DEBUG` block or `Debug.xcconfig`.
- `WatchInfo.plist` no longer sets `WKWatchOnly`; it names the iOS bundle id as its companion and
  runs independently (`WKRunsIndependentlyOfCompanionApp = YES`) (FR-011a, US2 acceptance 3+4).
- The watch honours a **companion-pushed** server override with no rebuild when the companion is
  present, and **falls back to the build-time default** when `isCompanionAppInstalled == false`,
  staying fully usable via QR device-login (FR-011).
