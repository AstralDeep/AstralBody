# Feature Specification: NSF TechAccess AI-Ready America Grant Writing Agent

**Feature Branch**: `011-nsf-grant-agent`
**Created**: 2026-05-01
**Status**: Draft
**Input**: User description: "Create a new agent: NSF TechAccess AI-Ready America Grant Writing Agent for the University of Kentucky. An expert grant-writing assistant focused on the NSF TechAccess: AI-Ready America State/Territory Coordination Hub solicitation NSF 26-508 Kentucky Coordination Hub proposal."

## Clarifications

### Session 2026-05-01

- Q: How should pre-decisional proposal draft text submitted to the agent be handled (provider posture, retention, logging)? → A: Same provider posture as every other agent in this app — no special handling beyond what the platform already enforces. Rely on existing platform controls.
- Q: Should the NSF 26-508 capability live as a new standalone agent or be merged into the existing `grants` agent? → A: Merge into the existing `grants` agent — the NSF 26-508 specialization is added to the current `grants` agent rather than created as a separate sibling module.
- Q: Cross-session memory of PI roster, partner list, and per-section draft state? → A: Inherit whatever cross-session memory posture the existing `grants` agent already has; no NSF-specific persistence is added.
- Q: Does the agent's scope cover only the State/Territory Coordination Hub, or also the related National Coordination Lead and AI-Ready Catalyst Award Competitions named in the solicitation? → A: All three. Scope is the full NSF 26-508 / TechAccess family — Coordination Hub (Kentucky), National Coordination Lead, and AI-Ready Catalyst Award Competitions. The agent helps with any of them when asked.
- Q: Per-response latency target for drafting requests? → A: Inherit existing platform latency expectations; no NSF-specific latency target is added.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Draft a section of the LOI or full proposal on demand (Priority: P1)

A member of the University of Kentucky proposal team opens the agent, names the section they are working on (e.g., "LOI synopsis," "Section 1 — Vision and Approach to Responsibilities," "Section 4 — Work Plan, Milestones, and Performance Metrics"), pastes any existing draft text, and asks the agent to produce a draft. The agent returns prose that uses the exact required NSF section heading (when applicable), addresses every required sub-element of that section, frames the Hub as a coordinator/convener rather than a direct service provider, grounds claims in Kentucky-specific detail (UK land-grant/Extension reach, KCTCS in all 120 counties, CPE statutory coordination role, COT cross-agency reach, etc.), maps training language to the literacy → proficiency → fluency continuum, and respects the 15-page narrative constraint by writing in compact prose rather than bullet-heavy lists.

**Why this priority**: This is the core, highest-volume use of the agent and the work product that the team most needs to produce against tight deadlines (LOI June 16, 2026; full proposal July 16, 2026; internal deadline ~July 9, 2026). Without on-demand drafting, the agent fails its primary purpose.

**Independent Test**: A team member requests a draft of any single proposal section, supplying minimal context. The agent returns a draft that (a) uses the exact required heading, (b) addresses each required sub-element listed in the solicitation for that section, (c) reflects coordinator/convener framing, and (d) cites at least one Kentucky-specific anchor point. The team member can immediately paste the output into the working proposal.

**Acceptance Scenarios**:

1. **Given** the user requests a draft of "Section 1 — Vision and Approach to Responsibilities" with no prior text, **When** the agent responds, **Then** the draft uses that exact heading, addresses all five Hub responsibilities, presents at least one statewide-coordination strategy and one approach to small-scale local pilots, and references at least one piece of UK/Kentucky prior experience with statewide multi-sector initiatives.
2. **Given** the user requests a one-page LOI synopsis, **When** the agent responds, **Then** the draft is sized for ~one page, opens with a working title that begins "Kentucky Coordination Hub:" (no acronyms), and reads as a compressed Section 1 + Section 2 narrative with a clear partner architecture and Hub responsibility coverage.
3. **Given** the user requests Section 4 (Work Plan, Milestones, and Performance Metrics) and provides no existing draft, **When** the agent responds, **Then** every NSF-required performance metric is named explicitly (individuals trained by category; small businesses and government entities assisted, with hours/dollars saved; statewide convenings and guidance facilitated; contributions to national best-practice repositories and sector coordination; AI Deployment Corps individuals trained and actively providing assistance; organizations provided technical assistance), and the draft also addresses reach, depth, and system-change metrics with a Year 1 baseline and a common cross-partner data-collection instrument.

### User Story 2 - Refine, strengthen, and gap-check an existing draft against NSF criteria (Priority: P2)

A team member pastes an existing draft section into the agent and asks it to (a) strengthen the language, (b) check it against NSF merit review criteria (Intellectual Merit, Broader Impacts) and the solicitation-specific review criteria, and/or (c) flag any required sub-element that is missing, weak, or off-tone. The agent returns specific, named gaps (not generic feedback), suggested rewrites for the weakest passages, and a short verdict on each applicable review criterion.

**Why this priority**: After the team has initial drafts, the highest-leverage activity is tightening them against the precise review rubric. Without this capability the team cannot self-review at the level reviewers will apply.

**Independent Test**: A team member submits a single draft section. The agent returns a list of missing or weak required sub-elements, at least one concrete rewrite of a weak passage, and an explicit assessment against each applicable review criterion. The reviewer can act on the feedback without further clarification.

**Acceptance Scenarios**:

1. **Given** a draft of Section 2 missing the governance/decision-making structure, **When** the user asks for a gap-check, **Then** the agent flags the missing governance content as a required sub-element and proposes draft language for it.
2. **Given** a draft of any section that overstates direct training delivery, **When** the user asks for a tone review, **Then** the agent flags the language as inconsistent with the Hub's coordinator/convener framing and rewrites it to emphasize backbone coordination and leveraging existing resources.
3. **Given** a draft of Section 4 that lists activities without baselines or evaluation, **When** the user asks for a strength check, **Then** the agent flags the absence of a Year 1 baseline, the absence of an independent evaluation component, and the absence of the literacy/proficiency/fluency mapping, and proposes language to address each.
4. **Given** the team's current section drafts collectively exceed the 15-page narrative limit, **When** the user asks "what do we cut to fit?", **Then** the agent produces a per-section table of current page allocation vs. recommended allocation, names which sub-elements are required (and therefore protected) vs. discretionary, and proposes an ordered cut list that respects all required sub-elements from the solicitation.

### User Story 3 - Produce supplemental materials and PI-facing logistics support (Priority: P3)

A team member asks the agent to produce supplemental documents and logistics artifacts: a PAPPG-format Letter of Collaboration template for a named partner, a draft Data Management Plan, a Mentoring Plan (only if the budget includes postdocs or graduate students), or a list of questions to send to the NSF program officer. The agent returns a ready-to-edit document that conforms to the solicitation's supplemental-materials rules (e.g., Letters of Collaboration only — no Letters of Support; no extra narrative documents).

**Why this priority**: These artifacts are required-but-bounded. They take less time than narrative drafting and become important closer to the internal deadline, after the narrative is largely settled.

**Independent Test**: A team member requests one supplemental artifact (e.g., a Letter of Collaboration template for KCTCS). The agent returns a single document that follows PAPPG format, names the partner's specific contribution to the Hub, and contains no language that the solicitation prohibits.

**Acceptance Scenarios**:

1. **Given** the user names a partner organization and its role, **When** the user requests a Letter of Collaboration template, **Then** the agent returns a PAPPG-conformant template that names the partner's specific contribution and avoids "letter of support" framing or content beyond what PAPPG permits.
2. **Given** the user requests a Data Management Plan, **When** the agent responds, **Then** the plan addresses data the Hub will collect across partners (training participation, business-assistance outcomes, convening attendance, evaluation data) and a common data-collection instrument across partners.
3. **Given** the user asks the agent to draft questions for the NSF program officer, **When** the agent responds, **Then** the questions are specific to the Kentucky Hub proposal context, are returned as a structured ready-to-send list (not a chat narrative), and avoid questions whose answers are explicit in the solicitation. Each generated question MUST have an internal "answer-in-solicitation?" check that filters out duplicates of solicitation content.
4. **Given** the user asks "what are the deadlines for this proposal?", **When** the agent responds, **Then** the response cites all three relevant dates — LOI due 2026-06-16 (Research.gov), full proposal due 2026-07-16 (Research.gov or Grants.gov; AOR signature required), and internal institutional deadline approximately 2026-07-09 — without requiring the user to be in an LOI drafting flow.

### Edge Cases

- The user asks the agent something unrelated to the NSF 26-508 proposal (e.g., "help me write a Python script," "summarize this paper"). The agent declines briefly and redirects the user back to the proposal.
- The user requests a "Letter of Support" or any supplemental document not permitted by the solicitation. The agent declines and explains the supplemental-materials restriction (only Letters of Collaboration, Data Management Plan, and — if applicable — Mentoring Plan are permitted).
- The user asks for help with a *different* NSF solicitation outside the TechAccess: AI-Ready America family. The agent declines and confirms its TechAccess scope (Coordination Hub + National Coordination Lead + AI-Ready Catalyst Award Competitions).
- The user asks for help on the National Coordination Lead Other Transaction Agreement or on a Catalyst Award Competition submission. The agent helps, but flags that those opportunities are competed and judged separately from the Kentucky Coordination Hub proposal — different mechanism, different rules, different deadlines — and asks the user to specify which opportunity they're drafting for so the agent applies the correct framing rather than carrying over Hub-specific language.
- The user requests a draft using a non-required section heading or omits one of the five required Section 1–5 headings. The agent uses the exact required heading text and notes that deviating from these headings results in return without review.
- The user's existing draft over-commits to direct training delivery, innovation outcomes, or economic-development promises beyond what coordination can credibly deliver. The agent flags the overpromise and proposes more grounded language.
- The user asks for activity-count metrics only. The agent supplies the NSF-required activity counts but additionally proposes reach, depth, and system-change metrics and an independent evaluation component, and reminds the user to establish a Year 1 baseline.
- The user gives only generic "AI training" language. The agent maps it explicitly to the literacy → proficiency → fluency continuum and asks who the audience is and which level of readiness is being targeted.
- The user has not yet identified all partners. The agent uses the likely Kentucky partnership architecture as a working assumption, names the assumption explicitly, and flags partnership gaps that the proposal will need to address.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: The NSF 26-508 capability MUST be delivered by extending the existing `grants` agent (rather than creating a new standalone agent). When the user invokes the existing `grants` agent for NSF TechAccess: AI-Ready America work, the agent MUST exhibit all behaviors specified here. Outside the TechAccess context, the existing `grants` agent's prior behaviors MUST be preserved; the TechAccess-specific scope-locking and redirect rules in this spec apply to TechAccess sessions and MUST NOT regress the agent's existing assistance for other grants work. The TechAccess scope spans the full NSF 26-508 / TechAccess: AI-Ready America family of opportunities — the State/Territory Coordination Hub (the Kentucky Coordination Hub proposal that is the primary use case), the National Coordination Lead (selected separately via Other Transaction Agreement), and the AI-Ready Catalyst Award Competitions (announced separately). The agent MUST redirect requests outside that family back to the proposal work.
- **FR-002**: The agent MUST be able to draft, on request, any of the following: the LOI title, the LOI synopsis (one page maximum), and each of the five required full-proposal sections — using the exact required headings: "Section 1 — Vision and Approach to Responsibilities," "Section 2 — Organizational Background, Team Expertise, and Partnership Rationale," "Section 3 — Current State of AI Planning and Coordination," "Section 4 — Work Plan, Milestones, and Performance Metrics," and "Section 5 — Resource Mobilization and Leveraging Additional Support."
- **FR-003**: For any draft of a full-proposal section, the agent MUST address every sub-element the solicitation lists for that section. For Section 1, this includes how the Hub executes all five Hub responsibilities, statewide coordination and scaling strategies, prior experience with statewide/multi-sector initiatives, and strategies for supporting small-scale local pilots. For Section 2, this includes the lead organization's statewide convening power and operational capacity, team qualifications and roles, partnership rationale, governance and decision-making structure, an Advisory Board, and known partnership gaps. For Section 3, this includes existing Kentucky AI readiness efforts and stakeholders, what is working and where gaps exist, and how the Hub accelerates progress. For Section 4, this includes major milestones and timelines per Hub responsibility, performance metrics for national dashboards, data collection and reporting mechanisms, and all NSF-required metrics. For Section 5, this includes plans for additional financial and in-kind support, and how additional resources expand reach and enable self-sustainment beyond NSF funding.
- **FR-004**: For any draft addressing the Hub's responsibilities, the agent MUST address all five required Hub responsibilities: AI Learning and Resource Navigator; State/Territory AI Readiness Strategic Plan; AI Deployment Support; AI Readiness Training and Capacity Building; and Coordination Within Priority Sectors.
- **FR-005**: The agent MUST frame the Hub as a coordinator and convener, not a direct service provider. Drafts MUST emphasize backbone coordination and leveraging existing resources rather than direct delivery of training or services. Language that overcommits to direct delivery or innovation outcomes MUST be flagged and rewritten.
- **FR-006**: For any training- or capacity-building-related language, the agent MUST map the language to the AI literacy continuum (Literacy → Proficiency → Fluency) and specify the audience and the targeted readiness level. Generic "AI training" language MUST be replaced with continuum-specific language.
- **FR-007**: For any draft of Section 4, the agent MUST include all NSF-required performance metrics by name: number of individuals trained (by category — educators, workforce, small business owners); number of small businesses and government entities assisted, with hours or dollars saved; number of statewide convenings held and guidance facilitated; contributions to national best-practice repositories and sector coordination activities; number of individuals trained for the AI Deployment Corps and actively providing assistance; and number of organizations provided technical assistance.
- **FR-008**: For any draft of Section 4, the agent MUST go beyond activity counts and propose reach metrics (sector, geography, demographics, prior AI exposure), depth metrics (pre/post literacy assessments, credential completions, AI tool adoption, ROI for assisted businesses), and system-change metrics (new partnerships, curriculum changes, policy changes, leveraged funding). The agent MUST require a Year 1 baseline before claiming progress, recommend a common cross-partner data-collection instrument, and recommend an independent evaluation component.
- **FR-009**: The agent MUST ground claims in Kentucky-specific detail wherever possible, drawing from the Kentucky partnership architecture (UK as land-grant institution with statewide Cooperative Extension; KCTCS reaching all 120 counties through 16 colleges; Kentucky Council on Postsecondary Education's statutory coordination role; Kentucky Governor's Office of Technology cross-agency relationships; Kentucky Cabinet for Economic Development; KentuckianaWorks and regional workforce boards; Kentucky SBDC; Kentucky Department of Education) and Kentucky priority sectors (healthcare, agriculture, advanced manufacturing, energy, education).
- **FR-010**: The agent MUST address equity considerations explicitly, including rural vs. urban access gaps (eastern Kentucky in particular), first-generation and adult learners, minority-owned small businesses, underserved school districts, and agricultural communities — and MUST connect equity claims to the trusted-messenger networks (e.g., Cooperative Extension, KCTCS) that make them deliverable.
- **FR-011**: The agent MUST be able to refine and gap-check existing draft text the user supplies: identify named gaps against required sub-elements, name gaps against NSF review criteria (Intellectual Merit, Broader Impacts, and the solicitation-specific criteria), suggest concrete rewrites of weak passages, and surface tone violations (especially overpromising and direct-delivery framing).
- **FR-012**: The agent MUST be able to draft supplemental materials limited to those the solicitation permits: PAPPG-format Letters of Collaboration for any named partner, a Data Management Plan, and a Mentoring Plan (only if the user indicates the budget includes postdocs or graduate students). The agent MUST refuse to draft Letters of Support or any supplemental narrative material the solicitation prohibits, and explain the restriction.
- **FR-013**: For any LOI title the agent generates, the title MUST begin with "Kentucky Coordination Hub:" and MUST NOT contain acronyms.
- **FR-014**: For any LOI synopsis the agent generates, the synopsis MUST fit within one page and MUST function as a compressed Section 1 + Section 2 narrative.
- **FR-015**: The agent MUST be able, on request, to produce questions to send to the NSF program officer, scoped to the Kentucky Hub proposal context and avoiding questions whose answers are explicit in the solicitation. This capability MUST be delivered by a dedicated tool whose output is a structured, ready-to-send list of questions (not a free-form chat answer), so refusal paths and "is the answer in the solicitation?" checks are deterministic and testable.
- **FR-016**: The agent MUST be able to advise on prioritization within the 15-page narrative limit, recommending what to compress or cut when total page budget is exceeded, and preserving required sub-elements over optional elaboration. This capability MUST be delivered either as a dedicated tool or as an explicit, testable mode within `gap_check_section`. The output MUST quantify current allocation against the 15-page budget and name which content is required vs. discretionary.
- **FR-017**: The agent's drafting style MUST be clear, direct prose rather than bullet-heavy proposal writing. Bullets are permitted only where the user explicitly requests them or where they materially improve readability of a list (e.g., partner roles, metrics).
- **FR-018**: The agent MUST emphasize sustainability beyond Year 3 in any Section 5 draft and in cross-section guidance, while avoiding overpromising on innovation, economic development, or outcomes that exceed what coordination can credibly deliver.
- **FR-019**: The agent MUST be able to align language with the Administration priorities cited in the solicitation, including the White House AI Action Plan, America's Talent Strategy, and executive orders on AI literacy and removing barriers to AI leadership, when the user requests such alignment or when drafting framing language. When the user explicitly requests Administration-priority alignment for a section draft, the resulting draft MUST contain at least one phrase drawn from the Administration-priority phrase list, and that presence MUST be verifiable by an automated check.
- **FR-020**: The agent MUST cite the critical deadlines on request: LOI due June 16, 2026 (Research.gov); full proposal due July 16, 2026 (Research.gov or Grants.gov, AOR signature required); internal institutional deadline approximately July 9, 2026. Deadline citation MUST be available both (a) as an automatic Alert inside LOI flows and (b) as a standalone tool call so the user can ask for the deadlines outside any drafting context.

### Key Entities *(include if feature involves data)*

- **Proposal Section**: A unit of the LOI or full proposal narrative the user is working on. Identified by its required heading (or "LOI synopsis" / "LOI title"). Has required sub-elements derived from the solicitation, a target page allocation under the 15-page total, and a current draft state (none, in-progress, refined).
- **Hub Responsibility**: One of the five non-negotiable responsibility areas the proposal must address. Each has its own framing constraints (e.g., "Training and Capacity Building" must be backbone coordination, not direct delivery).
- **Kentucky Partner**: A named institution or community organization with a defined contribution to the Hub (e.g., UK, KCTCS, CPE, COT, Cooperative Extension, KY Cabinet for Economic Development, KentuckianaWorks, KY SBDC, KDE). Used to ground claims and to seed Letter of Collaboration templates. The full partner roster is supplied by the user; the agent uses the likely partnership architecture as a working assumption when not specified.
- **Performance Metric**: A measurable outcome the proposal commits to track. Categorized as NSF-required activity counts, reach, depth, or system-change. Has a Year 1 baseline expectation.
- **Supplemental Artifact**: A document outside the 15-page narrative that the solicitation permits — Letter of Collaboration (PAPPG format), Data Management Plan, or Mentoring Plan (conditional on postdoc/grad-student budget). Letters of Support and any other narrative supplements are prohibited.
- **AI Readiness Level**: A point on the Literacy → Proficiency → Fluency continuum. Every training/capacity-building claim is anchored to one of these levels and to a specific audience.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: For any drafted full-proposal section, 100% of the required sub-elements listed in the solicitation for that section appear in the draft on the first pass.
- **SC-002**: For any drafted Section 4, 100% of the six NSF-required performance metric categories appear by name on the first pass, and the draft also includes at least one reach metric, one depth metric, and one system-change metric, plus a Year 1 baseline statement and an independent evaluation component.
- **SC-003**: At least 90% of substantive claims in any drafted section are anchored to a Kentucky-specific entity, network, sector, or geography (e.g., a named partner, a county-level reach claim, a priority-sector example), as judged by a reviewer on the proposal team.
- **SC-004**: 100% of off-topic requests are declined and redirected back to the proposal in a single response, without the agent producing off-topic content.
- **SC-005**: 100% of requests for prohibited supplemental artifacts (e.g., Letters of Support, additional narrative documents) are declined with an explanation of the supplemental-materials restriction.
- **SC-006**: For any training- or capacity-building-related passage produced or refined by the agent, 100% of training references specify both an audience and a literacy/proficiency/fluency level — no generic "AI training" references remain.
- **SC-007**: For any drafted full-proposal section, 0 instances of language that frames the Hub as a direct service provider rather than a coordinator/convener appear, as judged by a reviewer on the proposal team.
- **SC-008**: For any drafted LOI title, the title begins with "Kentucky Coordination Hub:" and contains no acronyms in 100% of cases.
- **SC-009**: For any drafted LOI synopsis, the synopsis fits within one page in 100% of cases and contains both Section 1-equivalent (vision/approach/responsibilities) and Section 2-equivalent (team/partners/governance) content.
- **SC-010**: A first-time user of the agent can produce a usable first draft of any single proposal section in under 10 minutes from initial prompt, including any clarifications they choose to provide.
- **SC-011**: Across the proposal team's full use of the agent through the July 9, 2026 internal deadline, the team produces complete first drafts of all five required sections, the LOI synopsis, the Data Management Plan, and Letter of Collaboration templates for every named partner, with no gap-check pass surfacing a missing required sub-element after the second iteration.
- **SC-012**: For any list of NSF program officer questions the agent produces, 0% of the generated questions have answers that are explicit in the solicitation text, as verified by an automated check at production time and by reviewer judgement at acceptance time.
- **SC-013**: For any page-budget prioritization request, 100% of the agent's recommended cuts preserve all required sub-elements (FR-003) for each section, and the recommendation includes a per-section current-vs-target page allocation table summing to ≤ 15 pages.
- **SC-014**: For any direct deadline question (e.g., "what's the full proposal deadline?"), 100% of responses cite all three relevant dates (LOI 2026-06-16, full proposal 2026-07-16, internal ~2026-07-09).

## Assumptions

- The agent assists a University of Kentucky proposal team. UK is the lead organization. PI, co-PIs, senior personnel, and the final partner roster are supplied by the user as needed; when not supplied, the agent uses the likely Kentucky partnership architecture (UK, CPE, COT, KCTCS, UK Cooperative Extension, KY Cabinet for Economic Development, KentuckianaWorks, KY SBDC, KDE) as a working assumption and names that assumption explicitly.
- The NSF TechAccess capability is delivered by *merging into the existing `grants` agent* rather than creating a new standalone agent module. The existing `grants` agent gains the TechAccess specialization described in this spec while retaining its prior charter for other grants work. The TechAccess-specific scope lock (decline off-topic / non-TechAccess requests within a TechAccess session) is engaged when the user is working on a TechAccess opportunity; it does not constrain the agent's behavior for other grants tasks routed to the same agent. The TechAccess scope covers the full NSF 26-508 / TechAccess: AI-Ready America family — State/Territory Coordination Hub (the Kentucky Coordination Hub proposal is the primary use case), National Coordination Lead (Other Transaction Agreement, selected separately), and AI-Ready Catalyst Award Competitions (announced separately). When a user requests help on the National Lead OTA or a Catalyst Award, the agent flags the differing mechanism, rules, and deadlines and asks which opportunity is being drafted so it applies the correct framing rather than reusing Hub-specific language.
- The full text of NSF 26-508 is treated as authoritative when its content conflicts with this specification or any user-provided draft language.
- The agent operates as an interactive assistant; it does not autonomously submit anything to Research.gov or Grants.gov.
- Data handling for pre-decisional draft text follows the same posture as every other agent in this application — no special provider, retention, or logging rules are imposed beyond the platform's existing controls. Draft text submitted to the agent is treated like any other agent input within AstralBody.
- The Mentoring Plan is conditional on the user confirming that the budget includes postdocs or graduate students; the agent does not assume one is required.
- Cross-session memory of PI roster, partner list, and per-section draft state inherits the existing `grants` agent's memory posture; no NSF-specific persistence is added. The user is expected to paste recurring context (PI roster, partner list, prior drafts) at the start of any session where it is needed.
- Per-response latency inherits the existing platform's expectations; no TechAccess-specific latency target is set. SC-010's "<10 minutes" remains a human-task metric, not an agent-response metric.
- "AI-Ready" is treated as a continuum (Literacy → Proficiency → Fluency) consistent with the solicitation's framing, and all training/capacity-building language is mapped to that continuum.
